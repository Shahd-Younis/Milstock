Slide 4 - Problem Statement
What is the core problem being addressed?
Food inventory operations are often managed through scattered spreadsheets, manual approvals, and delayed stock updates. This makes it difficult to track warehouse quantities, expiration dates, consumption, supplier requests, and accountability in one reliable system.

Who is affected?
Warehouse managers, kitchen/unit users, administrators, suppliers, and decision makers responsible for food supply availability and waste reduction.

Current pain points:
- Limited real-time visibility into stock levels across warehouses
- Manual request and approval workflows
- Difficulty tracking expiration, low stock, and waste
- Weak auditability for inventory movement and user actions
- Language and accessibility barriers for Arabic-speaking users

Why now?
Food operations need faster, more reliable digital systems that reduce waste, improve transparency, and support data-driven decisions.

Real-world impact if unsolved:
Stock shortages, over-ordering, expired items, delayed approvals, inaccurate reports, and higher operational cost.


Slide 5 - Objectives & Project Scope
SMART Objectives:
1. Build a full-stack inventory platform that manages food products, warehouses, users, suppliers, requests, and reports.
2. Provide role-based dashboards for administrators, kitchen/unit users, and supplier-related workflows.
3. Support bilingual English LTR and Arabic RTL interfaces for wider usability.
4. Enable real-time API-based data operations using MongoDB and reusable frontend API services.
5. Deploy the frontend and backend as separate cloud services for public demo access.

What's In Scope:
- User authentication and JWT-based protected routes
- Admin and unit/kitchen dashboards
- Inventory item CRUD and item details
- Warehouse locations and warehouse stock records
- Supplier and request/order workflows
- Consumption, waste, movement logs, notifications, audit logs, and reports
- CSV export with Arabic Excel support
- Responsive desktop and mobile interface
- Frontend deployment on Vercel and backend deployment on Render

Out of Scope:
- Native mobile apps
- Barcode/RFID hardware integration
- Payment or procurement finance modules
- AI demand prediction in the current version


Slide 6 - Proposed Solution
MilStock is a bilingual web-based food inventory and warehouse management system.

It centralizes food supply tracking, warehouse stock, requests, suppliers, users, notifications, waste analysis, and operational reports in one platform.

Core capabilities:
- Admin dashboard for system-wide monitoring
- Kitchen/unit user dashboard for request and consumption workflows
- Warehouse-level stock visibility
- Low-stock and expiration monitoring
- Supplier and order tracking
- Audit logs and CSV exports
- English and Arabic interfaces

How it improves the current process:
- Replaces fragmented manual tracking with a connected digital workflow
- Gives users role-specific access instead of a one-size-fits-all interface
- Makes stock and request data easier to search, filter, export, and audit
- Supports Arabic RTL for operational teams that need localized workflows

Unique value proposition:
A practical food-only inventory platform designed around warehouse operations, bilingual usage, and real deployment rather than a generic stock demo.

Slide 9 - System Requirements
Functional Requirements:
FR1: The system shall allow users to register, log in, and access protected routes based on role.
FR2: The system shall allow admins to manage users, warehouses, suppliers, products, and stock records.
FR3: The system shall allow kitchen/unit users to create and track supply requests.
FR4: The system shall support inventory search, filtering, item details, and stock status indicators.
FR5: The system shall track consumption, waste, movement logs, notifications, and audit logs.
FR6: The system shall export selected data to CSV with Arabic Excel compatibility.
FR7: The system shall support English and Arabic interfaces.

Non-Functional Requirements:
Performance: API-driven pages should load efficiently with reusable resource hooks and filtered datasets.
Security: JWT authentication, hashed passwords, role-based access, CORS allowlist, and secure headers.
Scalability: Separate frontend, backend, and database services allow independent deployment and scaling.
Availability: Cloud deployment supports public access for demos and future operation.
Usability: Responsive dashboard design supports desktop and mobile use.
Maintainability: Modular routes, controllers, models, and reusable frontend components.


Slide 10 - Workflow / Use Case Diagram
Main Actors:
- Admin
- Kitchen / Unit User
- Supplier User
- System

Core Workflow:
1. User logs in
2. System verifies JWT and role
3. Admin manages warehouses, users, products, and suppliers
4. Unit user creates a supply request
5. Admin reviews the request
6. Admin approves or rejects the request
7. If approved, stock movement / fulfillment is recorded
8. System updates warehouse stock and logs the action
9. Notifications and audit records are generated
10. Reports show consumption, waste, inventory status, and operational trends

Diagram Suggestion:
Use a left-to-right flow:
Login -> Role Dashboard -> Request / Inventory Action -> Backend API -> MongoDB -> Notification / Audit Log -> Report

Slide 13 - Project Impact & Innovation
Real-world problem solved:
MilStock helps food operations maintain accurate stock visibility, reduce waste, manage requests, and improve accountability across warehouses and kitchens.

Value delivered:
- Operational: Faster request handling and clearer warehouse visibility
- Economic: Better stock control can reduce over-ordering and expired inventory
- Social: Supports reliable food supply operations for teams and beneficiaries
- Technical: Demonstrates a real deployed full-stack system with bilingual support

Innovation:
MilStock is not a generic inventory tracker. It focuses specifically on food inventory workflows, expiration monitoring, warehouse distribution, consumption, waste, and Arabic/English operational use.

Alignment with digital transformation:
The project supports digital recordkeeping, cloud deployment, data-driven reporting, and accessible localized software, aligning with modern public and private sector digital transformation goals.

Scaling potential:
The system can evolve into a multi-branch warehouse platform with barcode scanning, demand forecasting, supplier portals, and integration with procurement systems.
